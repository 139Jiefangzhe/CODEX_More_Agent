# 前端 Bento 2.0 重构执行计划（2026-03-31）

更新时间：2026-03-31（Asia/Shanghai）

## 1. 背景与目标

本次重构目标：将 Dashboard 从“传统卡片后台”升级为“面向 Multi-Agent 监控的 Bento 2.0 + Glassmorphism 2.0 控制台”，并保持可维护性、可扩展性与暗黑模式优先体验。

核心设计方向：

- 布局：Bento 不对称模块化网格（提升扫描效率）
- 视觉：Liquid Glass + Soft UI 轻深度（提升触感与层次）
- 交互：状态可视化、微交互、响应式折叠、可访问性增强
- 工程：设计 Token 化、页面模块化、路由懒加载、低回归上线

## 5. 风险与缓解

1. 视觉改造过大导致交互回归
- 缓解：仅做表现层与布局重构，不改 API 契约和业务流程

1. 图表颜色在暗黑模式下对比不足
- 缓解：统一颜色 token 并在图表组件中显式指定

1. 路由懒加载引入加载闪烁
- 缓解：使用玻璃态骨架/占位容器作为 Suspense fallback
